# 项目重构计划

> 目标:按 Vue 官方风格指南,修正「页面级组件放进全局组件目录」「文件/目录命名不统一」两类问题。
> 范围约定:**目录统一 PascalCase**;**本次不改 store / util 文件后缀**(留待后续)。

---

## 一、现状问题

### 1. 页面级组件混在 `components/` 里
- `components/Test/MessageTester.vue`、`components/Test/WebSocketTester.vue` 在 `router/index.js:61`、`:110` 被当作**路由页面**直接挂载,物理上却在组件目录。页面应归 `views/`。

### 2. 根目录散落业务组件(无 feature 归属)
`components/` 根下:`TokenManager.vue`、`GameStatus.vue`、`ServerRoleList.vue`、`ChangelogCard.vue`、`ClubCarKing.vue`、`CarTaskCard.vue`。
- `ClubCarKing.vue` 属于 `Club/`(仅被 `GameStatus.vue` 引用)。
- `CarTaskCard.vue` **全仓库无任何引用**(模板与 import 均搜不到),疑似死代码。

### 3. 文件命名不符合 PascalCase 完整单词规范
| 现状 | 改为 |
|---|---|
| `Club/ClubWarrank.vue` | `Club/ClubWarRank.vue` |
| `Club/Signin.vue` | `Club/SignIn.vue` |
| `Club/Rank.vue` | `Club/ClubRank.vue`(避免泛化单词) |
| `cards/FightPvp.vue` | `Cards/FightPvP.vue` |
| `cards/Unlimitedlineup.vue` | `Cards/UnlimitedLineup.vue` |

### 4. 目录大小写不一致
`components/cards/` 为小写,其余 `Club/ Common/ Daily/ Team/ Tower/ Setting/ Test/` 均 PascalCase。

### 5. `cards/` 是「杂物抽屉」
20 个文件混放(战斗 / 钓鱼 / 捞瓶子 / 排行榜 / 升星等不同 feature),无按功能聚合。

### 6. 路由策略未收敛:自动路由与手写路由重复注册(**最高优先级,先于一切文件移动**)
当前 `unplugin-vue-router` 以 `src/views` 为源(`vite.config.js:65`),已扫描出**所有顶层 views 页面**(见 `typed-router.d.ts:21-38`:`/Dashboard` `/GameFeatures` `/DailyTasks` `/BatchDailyTasks` `/LegionWar` `/Profile` `/Home` 等),而非文档原先以为的「仅 TokenImport」。

与此同时 `generatedRoutes` 被**插入了两处**:
- `router/index.js:104` — 塞进 `/admin` 的 `children`
- `router/index.js:130` — 又挂到根路由

这些 `views/*` 页面**已经被自动+手写双重(乃至三重)注册**:例如 Dashboard 同时存在手写 `/admin/dashboard`、自动 `/Dashboard`、以及被塞进 `/admin` children 的自动副本。后续任何「移动页面 / 删两行 import」的操作都会在此基础上放大重复,必须**先把路由策略定下来**再动文件。

## 四、本项目特有约束(迁移必读)

- **`unplugin-vue-components`** 自动扫描 `src/components`,按**文件名**注册全局组件 → 改文件名必须同步改所有模板里的 `<标签名>`;移动目录不影响自动注册,但会断显式 `import` 路径。
- **`unplugin-vue-router`** 以 `src/views` 为路由源,排除 `**/components/**`、`**/test**.vue`、`**/**Modal.vue`(`vite.config.js:67`)。注意:**它已扫描全部顶层 views 页面并自动生成路由**,且 `generatedRoutes` 当前被挂载两次(`router/index.js:104`、`:130`)。Test 组件移入 `views/Tools/` 时,因文件名 `MessageTester`/`WebSocketTester` 不以 `test` 开头、不命中 `**/test**.vue` 排除,会**额外生成路由**。这不是「删两行 import」能解决的,须先按阶段 1 收敛路由策略。
- **`@components` / `@views`** 别名 + 显式 import 路径,随移动同步更新。
- **macOS 大小写不敏感**:目录改大小写须两步 `git mv`(如 `git mv cards _cards_tmp && git mv _cards_tmp Cards`),否则 git 不记录变更。

## 五、迁移阶段(按安全顺序,每阶段一个 commit)

> ⚠️ 顺序调整:**路由策略收敛(阶段 1)必须先于一切文件移动**。在 `generatedRoutes` 重复挂载未解决前移动页面,只会放大重复路由问题。

**阶段 0 — 清理**
确认并删除死代码 `components/CarTaskCard.vue`。
```bash
rg --files src | rg "CarTaskCard"      # 查文件名(确认文件存在)
rg "CarTaskCard|<CarTaskCard" src      # 查引用:模板标签 + import,应 0 命中(已验证)
```

**阶段 1 — 收敛路由策略(新增,最高优先级)**
先决定项目的路由来源,二选一并贯彻:
- **方案 A(推荐):收敛到手写路由。** 最稳妥的做法是**直接停用 `unplugin-vue-router`**:在 `vite.config.js` 不再注册 `routerPlugin`(或不传 `routesFolder`),同时删掉 `router/index.js:2` 的 `import * as autoRoutes`、`:6` 的 `generatedRoutes` 定义,以及 `:104`/`:130` 两处 `...generatedRoutes` 插入和 `:154` 的 `handleHotUpdate`。这样路由 100% 来自手写 `my_routes`,路径(`/admin/*` 前缀、`requiresToken` meta、`DefaultLayout` 嵌套)完全可控,与现有导航守卫一致。
  > 备选:若想保留插件但屏蔽自动路由,可尝试在 `exclude` 增加 `**/views/**` 让 `generatedRoutes` 为空——但此 glob 行为**需实测确认**(`typed-router.d.ts` 应重新生成为仅剩类型声明、无新增路由),不如直接停用插件可靠。
- **方案 B:收敛到文件路由。** 删除手写的页面路由,统一用自动路由,并通过 `definePage()` 补 `meta`/`layout`。改动面更大,需重做 `/admin` 嵌套与守卫。

无论哪种,**必须保证 `generatedRoutes` 不再被挂载两次**(当前 `router/index.js:104` 与 `:130` 同时插入)。
```bash
rg "generatedRoutes" src/router/index.js      # 方案A:应 0 处;方案B:保留 1 处
rg "autoRoutes|auto-routes" src/router/index.js
```
验收:`npm run build` 后逐一访问 `/admin/dashboard`、`/admin/game-features`、`/admin/daily-tasks` 等,确认每个页面只解析到唯一路由、无重复 name 警告。

**阶段 2 — 纯改名**(最易出错)
`Signin→SignIn`、`ClubWarrank→ClubWarRank`、`FightPvp→FightPvP`、`Unlimitedlineup→UnlimitedLineup`、`Rank→ClubRank`。
用 `git mv` + 同步改**模板标签**与 import 路径(自动注册按文件名,改名必断旧标签)。
```bash
# 改名前先列出所有引用点(模板标签 + import),逐一核对
rg "ClubWarrank|Signin|FightPvp|Unlimitedlineup" src
rg "components/Club/Rank|<Rank\b|\bRank\b" src   # Rank 泛名,需人工确认命中
```

**阶段 3 — 页面归位**
`components/Test/*` → `views/Tools/*`。
注意:`MessageTester.vue`/`WebSocketTester.vue` 文件名**不以 `test` 开头**,不命中 `vite.config.js:67` 的 `**/test**.vue` 排除 → 移入 `views/` 后会被自动生成路由。须按**阶段 1 选定的方案**处理(方案A:在 exclude 补 `**/Tools/**` 或保留手写路由并改 `@/views/Tools/*`;方案B:删手写路由用自动路由)。
```bash
rg "Test/MessageTester|Test/WebSocketTester|message-test|websocket-test" src
```

**阶段 4 — 根组件归类**
`ClubCarKing→Club/`、`TokenManager + ServerRoleList→Token/`、`GameStatus→Status/`、`ChangelogCard→Changelog/`。
移动目录不影响自动注册(按文件名),但会断**显式 import**;`GameStatus` 等被模板自动引用的组件(如 `views/GameFeatures.vue:32` 的 `<GameStatus />`)标签名不变,无需改。
```bash
rg "ClubCarKing|TokenManager|ServerRoleList|GameStatus|ChangelogCard" src
rg "@/components/(ClubCarKing|TokenManager|ServerRoleList|GameStatus|ChangelogCard)" src   # 显式 import,必改
```

**阶段 5 — 拆分 `cards/`**
先 `cards/ → Cards/`(macOS 两步 `git mv`:`git mv cards _cards_tmp && git mv _cards_tmp Cards`),再拆 `Helper/ Rank/ Upgrade/ Activity/` 子目录;同步更新所有 import 路径。
```bash
rg "@/components/cards|components/cards" src   # 显式 import 路径,逐一更新
```

**阶段 6 — 目录大小写收尾**
核对其余目录均为 PascalCase,补齐遗漏。

**每阶段验收**
```bash
npm run lint
npm run build
```
确认 `typed-router.d.ts`、`components.d.ts` 重新生成且无报错;手动点开受影响页面验证 UI。
