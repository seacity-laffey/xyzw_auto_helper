# xyzw_auto_helper 项目功能与文件分析

生成时间：2026-06-22

## 1. 项目定位

该项目是一个基于 Vue 3 + Vite 的 XYZW/咸鱼游戏 Web 辅助平台。核心目标不是传统账号登录，而是围绕游戏 Token 管理、bin/URL/二维码导入、WebSocket 长连接、BON 协议编解码和游戏命令自动化，提供单账号和批量账号的日常任务、活动、俱乐部/军团、排行榜、爬塔、车辆、答题、梦境等辅助能力。

## 2. 技术栈与运行入口

- 前端：Vue 3、Vite、Pinia、Vue Router、Naive UI、Arco CSS、UnoCSS、SCSS。
- 通信：浏览器 WebSocket + 自定义 BON 二进制协议，支持 lx/x/xtm 加密通道。
- 存储：localStorage、IndexedDB、VueUse useLocalStorage。
- 自动化：tokenStore 统一发送游戏命令，DailyTaskRunner 和 src/utils/batch/* 负责任务编排。
- 部署：Vite 静态构建、Docker/Nginx、Cloudflare Pages Worker、Azure Static Web Apps 配置。
- 入口文件：`index.html` -> `src/main.js` -> `src/App.vue` -> `src/router/index.js`。

常用命令来自 `package.json`：

```bash
npm run dev      # Vite 开发服务器
npm run build    # 生产构建
npm run preview  # 预览构建产物
npm run lint     # ESLint 自动修复 src 下 vue/js/ts
npm run format   # Prettier 格式化 src 下源码
npm run testr    # test/roleToken.js
npm run testd    # test/token.js
```

## 4. 目录结构说明

- `src/views`：路由页面，组合业务组件并处理页面级交互。
- `src/components`：可复用业务组件，按 Daily、Club、Team、Tower、cards、Test、Common 分组。
- `src/stores`：Pinia 和事件总线状态层，tokenStore 是核心。
- `src/utils`：协议、WebSocket、任务编排、导出、排行榜、日期、静态字典等工具。
- `src/utils/batch`：批量任务的独立任务工厂和调度工具。
- `src/xyzw`、`source`：逆向/参考代码，不直接作为 Vue 页面入口。
- `public`：直接发布的静态资源和题库。
- `docker`、`worker.js`、`wrangler.toml`、`staticwebapp.config.json`：部署与边缘代理。
